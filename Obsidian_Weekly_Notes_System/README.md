# Obsidian + Markdown 按周工作笔记体系（跨周任务友好）

这是一套以“周”为主驾驶舱、以“项目”为跨周任务栖息地的笔记体系。
核心原则：

- **周笔记负责规划与推进**：本周要做什么、怎么做、做到了哪。
- **项目笔记负责承载跨周任务**：一个任务/项目无论持续几周，都在同一处维护事实与进度。
- **月笔记负责复盘与资源配置**：从更高视角调整目标、节奏与投入。

## 目录建议

本套模板仅提供内容结构，你可以按需创建以下目录（或对齐你已有的目录命名）：

- `00_Inbox/`：临时收集（碎片、截图、口头需求）
- `10_Daily/`：日记（执行、会议速记）
- `20_Weekly/`：周记（核心管理层）
- `30_Monthly/`：月记（复盘与规划）
- `40_Projects/`：项目主页（跨周任务的唯一真理来源）
- `99_Templates/`：模板

本目录内已提供：

- `Templates/`：日/周/月/项目模板
- `Examples/`：示例文件

## 关键做法：让任务跨周而不失控

### 1) 任务必须有“归属地”

任何预计会跨周的事项，都要创建或归入某个项目主页（`40_Projects/`）。

- 周笔记里只出现“本周要推进的项目切片”
- 项目笔记里维护：目标、里程碑、任务分解、当前状态、阻塞与决策

### 2) 用“任务 ID”避免重复与歧义（可选但强烈推荐）

给项目内重要任务一个稳定 ID（例如 `T-2026-001`）。

- 项目笔记里：任务列表以 ID 为主键
- 周笔记里：引用 ID + 链接到项目（或任务块）

好处：跨周转移时只是把引用搬到下一周，任务本体仍在项目里。

### 3) 周→日：从周笔记领办当天

每天早上从周笔记的“本周计划”里挑选 1–3 条放进日记的“今日 TOP3”。
日记的任务尽量链接回周笔记/项目笔记，避免出现“孤儿任务”。

### 4) 周复盘：只做两件事

- **完成的**：在项目里勾选完成（或更新状态），周记留痕即可
- **未完成的**：明确去向：
  - 仍是该项目的一部分：留在项目 Backlog，并在下周周记重新引用
  - 不再重要/不做了：在项目里标注取消原因（避免反复出现）

## 推荐的笔记链接习惯

- 周记顶部链接：`[[2026-01]]`（当月）、`[[2026-W03]]`（上周）、`[[2026-W05]]`（下周）
- 项目主页固定字段：`状态 / 负责人 / 干系人 / 目标 / DoD / 当前阻塞`
- 周记里维护一个“活跃项目表”，只列本周在动的 3–7 个项目

## 关于模板变量

模板中的 `{{date:...}}` 写法对齐 Obsidian 常见的周期笔记/模板插件生态（如 Periodic Notes + 模板功能）。
如果你使用的模板引擎语法不同，只需要把日期占位符替换成你的语法即可。
