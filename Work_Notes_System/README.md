# Obsidian 工作笔记管理系统指南

这个目录包含了一套基于 **周维度** 和 **项目维度** 的工作笔记管理系统。

## 📂 目录结构说明

- **00_Inbox**: 临时存放处，所有未经整理的截图、速记都先放这里，每天或每周清空一次。
- **10_Daily**: 每日笔记 (`YYYY-MM-DD`)。用于记录当天的流水账、会议纪要和临时任务。
- **20_Weekly**: 每周笔记 (`YYYY-Www`)。这是**核心管理层**。用于规划本周重点，连接项目和每日行动。
- **30_Monthly**: 月度复盘 (`YYYY-MM`)。用于宏观视角的规划和回顾。
- **40_Projects**: 项目主页。**跨周任务的栖息地**。所有耗时超过一周的任务都应建立一个独立的项目文件。
- **99_Templates**: 存放笔记模板 (Daily, Weekly, Monthly, Project)。

## 🚀 核心工作流 (Workflow)

### 1. 应对跨周任务 (Projects)
不要在周报里只写 "做项目A"。
1. 在 `40_Projects` 中新建文件 `Project_A.md`。
2. 在里面列出所有步骤（Task 1, Task 2, Task 3...）。
3. 无论这个项目持续多少周，`Project_A.md` 都是它的**唯一真理来源**。

### 2. 每周规划 (Weekly Planning)
每周一早上（或周日晚上）：
1. 新建本周笔记（如 `2026-W04`）。
2. 打开 `Project_A.md`，挑选出**本周**能做的任务（如 Task 1, Task 2）。
3. 将这些任务写到 `2026-W04` 的 "活跃项目" 区域，并链接到项目文件 (`[[Project_A]]`)。
   - *Tip: 可以在 Obsidian 中使用引用块或任务嵌入。*

### 3. 每日执行 (Daily Execution)
每天早上：
1. 打开 `Daily Note`。
2. 或者是直接看 `Weekly Note`，从中领办今天的任务。
3. 记录当天的突发事项到 `Inbox` 或 `Daily Note`。

### 4. 每周复盘 (Weekly Review)
每周五下午：
1. 检查 `2026-W04` 中的任务完成情况。
2. **未完成的任务**：
   - 还没做？-> 决定是推迟到下周 (`W05`) 还是放回 `Project_A` 的待办列表。
   - 已完成？-> 去 `Project_A` 里把总进度勾选上。

## 🛠️ 如何开始
1. 安装 [Obsidian](https://obsidian.md/).
2. 安装 **Periodic Notes** 插件（可选，但推荐），配置模板路径指向 `99_Templates`。
3. 开始创建你的第一个项目和本周笔记！
